安卓四大组件

Activity
  • 解释
    • 用户与应用程序交互的窗口。一个Activity通常对应一个单独的视图，相当于Web中的网页。
    • 一般情况下，该界面窗口会填满整个屏幕，但是也可以比屏幕小，或者浮在其他的窗口之上。
    • 一个 Android 应用程序通常由多个 Activity 组成，但是其中只有一个为主 Activity。其作用相当于Java应用程序中的 main 函数，当应用程序启动时，作为应用程序的入口首先呈现给用户。
  • 使用
    • 开启与关闭：当新打开一个activity时，之前的视图会被压栈保存，回退操作可返回以前打开过的视图。
    • 生命周期：产生，运行，销毁。过程中会使用到许多方法，例如创建onCreat(),激活onStart(),恢复onResume(),暂停onPause(),停止onStop(),销毁onDestroy()等、
  • 状态
    • 运行态：Activity 运行于屏幕的最上层并且获得了用户焦点
    • 暂停态：当前 Activity 依然存在，但是没有获得用户焦点。换言之，不在最上层，有部分能被用户看见
    • 停止态：当前 Activity 完全被处于运行态的 Activity 遮挡住，其用户界面完全不能被用户看见。处于停止态的 Activity 依然存活，也保留了自己所使用的内存和用户信息，但是一旦系统缺乏资源，停止态的 Activity 就会被杀死以释放资源。
  • 回调方法与使用：
名称 调用时间
onCreate(Bundle savedInstanceState) 创建 Activity 时调用。 设置在该方法中，还以 Bundle 的形式提供对以前储存的任何状态的访问。其中参数 savedInstanceState 对象是用于保存 Activity 的对象的状态。
onStart() Activity 变为在屏幕上对用户可见时调用。
onResume() Activity 开始与用户交互时调用（无论是启动还是重启一个活动，该方法总是被调用）。
onPause() 当 Android 系统要激活其他 Activity 时，该方法被调用，暂停或收回 CPU 和其他资源时调用。
onStop() Activity 被停止并转为不可见阶段时调用。
onRestart() 重新启动已经停止的 Activity 时调用。
onDestroy() Activity 被完全从系统内存中移除时调用。 该方法被调用可能是因为有人直接调用 finish() 方法或者系统决定停止该活动以释放资源。
  • 生命周期

Service
  • 解释
    • 类似Activity，但是没有视图的程序，没有用户界面，可以在后台运行很长时间，相当于操作系统中的一个服务。
  • 分类：
    • 本地Service:只能由承载该Service的应用程序访问的组件
    • 远程Service:供在设备上运行的其他应用程序远程访问的Service
  • 使用
    • 启动:Context.startService(Intent service)
    • 绑定:Context.bindService()
BroadcastReceiver

  • 解释

    ​ 用来接收来自系统和其他应用程序的广播，并作出回应。例如开机完成，系统产生广播，使得有开机启动服务的软件开始工作；不能生成UI

  • 使用

    • 通过NotificationManager通知用户有事件发生

  • 注册方式

    • 在AndroidMainfest.xml中进行静态注册
    • 在运行时，代码中使用ContextregisterReceiver()进行动态注册
    • ps：只要注册了，即使对应事件的广播来临时，该程序并未启动，系统也会自动启动该应用程序来对事件进行处理。// 有点像手机关机后，当定的闹钟时间到达时，手机会自动开机，启动时钟这个应用程序？ 查询后，手机关机后，闹钟依旧会响的原因是，关机并不是真正的完全关闭，而是有一部分电量在保持着部分功能在运行。
ContentProvider

  • 解释

    实现两个程序之间数据的交换或共享。

  • 使用

    • ContentProvider实现了一组标准方法的接口，从而能够让其他的应用保存或读取ContentProvider提供的各种数据类型
Intent

  • 解释

    ​ Intent并不是四大核心组件。Intent是对即将进行的操作的抽象描述，承担了Android应用程序三大核心组件（A，S，B）相互之间的通信功能

布局管理

线性布局
  • 水平或者垂直方向排列控件
  • 主要属性：
    • android: gravity： 内部空间显示位置，对于空间本身而言
    • android: orientation： (horizontal / vertical )
    • android: latout_weight ：所占权重
约束布局
帧布局
  • 以层叠的方式显示
  • 主要属性：
    • android: layout_gravity 控件位于布局的方向，相对于父控件而言的
    • android: visibility (View.VISIBLE/View.INVISIBLE/View.GONE) 设置控件是否可见
表格布局
  • 表格形式排列空间，每个单元格都可以添加控件
  • 表格布局需要和TableRow配合使用，每一行都是一个TableRow
  • 主要属性：
    • android: stretchColumns 该列被拉伸
    • android: shrinkColumns 该列被收缩
    • android: collapseColumns 该列被隐藏
  • 控件属性：
    • android: layout_column 显示该单元显示位置
    • android: layout_span 设置该单元占据几列，默认为1列

简单控件

TextView
  • 用于显示文字
EditText
  • 可以进行编辑操作的文本框
Button
  • 设置点击事件
  • 监听button：button.setOnClickListener(){}
RadioButton
  • 单选按钮，与RadioGroup配合使用，提供多个互斥选项集
  • 监听RadioButton：setOnCheckedChangeListener(){}
ImageView
  • 显示图像，提供图像编辑
  • 监听：imageView.setImageResource();
